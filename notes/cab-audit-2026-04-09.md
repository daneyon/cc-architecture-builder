## Key Observations

### Plugin-First Architecture: PASS

The Phase 2 migration successfully corrected component locations:

- `plugin.json` clean (no stale custom paths)

This validates both the Phase 2 structural migration AND the Phase 1 audit skill plugin detection logic.

### "Cobbler's Children" Pattern

CAB is a comprehensive CC architecture framework that documents best practices for hooks, rules, security, and agent configuration — but its own project implements fewer of these than it recommends. This is the primary source of findings:
- Documents 26 hook events but uses 0
- Documents project rules but has no `.claude/rules/`
- Documents agent frontmatter best practices but uses invalid fields
- Documents seed instruction patterns but CLAUDE.md still has verbose sections

### Agent Frontmatter: Persistent LL-15 Gap

LL-15 was recorded 2026-04-08 documenting that `context:` is not a valid agent field, yet 3/4 agents still use it. This gap persisted through the Phase 2 migration because Phase 2 focused on structural paths, not frontmatter field validity. The `disallowedTools` and `permissionMode` issues are related — stale field patterns from pre-R2 development.
